<----- NOTAS DEL PROYECTO ----->
 
Vamos a comenzar con el ultimo proyecto de Diseño web responsive de freeCodeCamp, y asi obtendremos la certificacion.

El ultimo proyecto es una pagina de "Portafolio personal" vamos a usar las condiciones que nos ofrece freeCode para realizar la pagina, que son las siguientes:

Objetivo: Crea una aplicación que sea funcionalmente similar a https://personal-portfolio.freecodecamp.rocks

1-■ Tu portafolio debe tener una sección de bienvenida con un id de welcome-section

    Solucion: Como siempre vamos a crear 3 etiquetas principales (header, main y footer), dentro del main vamos a crear un section con id=welcome-section.

2-■ La sección de bienvenida debe tener un elemento h1 que contenga texto

    Solucion: dentro de #welcome-section vamos a colocar un h1, que diga algo asi como "Soy Lucas", y abajo un h2 que diga: "Soy desarrollador de sitios web".

3-■ Tu portafolio debe tener una sección de proyectos con un id de projects

    Solucion: Vamos a crear otro section con id=projects.

4-■ La sección de proyectos debe tener al menos un elemento con una class llamada project-tile para mantener el proyecto

    Solucion: Vamos a crear al menos un div con la class .project-title que contendra nuestro proyecto creado anteriormente. (puedo colocar todos los que hice anteriormente).

5-■ La sección de proyectos debe contener al menos un enlace que redirija a un proyecto

    Solucion: Dentro del div .project-title vamos a colocar un a que nos redirija a la web de nuestro proyecto.

6-■ Tu portafolio debe tener una barra de navegación con un id de navbar

    Solucion: Crearemos en el header, un elemento nav con un id=navbar.

7-■ La barra de navegación debe contener al menos un enlace en donde puedas hacer clic para navegar a diferentes secciones de la página

    Solucion: Crearemos al menos un enlace en la #navbar que nos desplace dentro de la misma pagina.

8-■ Tu portafolio debe tener un enlace con un id de profile-link, el cual abre tu GitHub o el perfil de freeCodeCamp en una pestaña nueva

    Solucion: Podemos crear una segunda seccion para hacer que nos contacten, ahi colocaremos al menos un a que tenga un id=profile-link, y que nos redirija a nuestro perfil de GitHub en una pestaña nueva (con target=_blank).

9- Tu portafolio debe tener al menos una consulta de medios

    Solucion: Cuando hagamos la pagina responsive, vamos a tener que usar media queries, asi que se resolveria este punto.

10- La altura de la sección de bienvenida debe ser igual al viewport

    Solucion: El height de la #welcome-section va a ser de 100vh.

11- La barra de navegación debe siempre estar en la parte superior del viewport

    Solucion: El header va a tener un position=fixed en top=0 left=0.
